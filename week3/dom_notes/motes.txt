# 🌳 The DOM (Document Object Model): Essential Notes

> The DOM is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. The DOM represents the document as nodes and objects; that way, programming languages can interact with the page.

---

## 🔍 1. Selecting Elements
Before you can manipulate an element, you need to grab it from the DOM.

**Modern Methods (Recommended):**
- **`document.querySelector(selector)`**: Returns the *first* element that matches a CSS selector.
- **`document.querySelectorAll(selector)`**: Returns a NodeList of *all* elements matching a CSS selector.

**Traditional Methods:**
- **`document.getElementById('id')`**: Selects a single element by its unique ID.
- **`document.getElementsByClassName('class')`**: Returns an HTMLCollection of elements with the specified class.
- **`document.getElementsByTagName('tag')`**: Returns an HTMLCollection of elements with the specified tag name.

```javascript
// Selecting by CSS class
const mainTitle = document.querySelector('.title');

// Selecting multiple elements
const listItems = document.querySelectorAll('li');

```

---

## 🛠️ 2. Manipulating Elements

Once you have selected an element, you can change its content, attributes, and styles.

### 📝 Changing Content

* **`.textContent`**: Gets or sets the raw text content of a node. (Safest and fastest).
* **`.innerHTML`**: Gets or sets the HTML markup contained within the element. (Use with caution due to security risks like XSS).

```javascript
const paragraph = document.querySelector('.desc');
paragraph.textContent = "This is the updated text!";

```

### 🎨 Changing Styles and Classes

Directly modifying styles is possible, but toggling CSS classes is generally best practice.

* **`.style`**: Accesses inline styles.
* **`.classList.add('class')`**: Adds a class.
* **`.classList.remove('class')`**: Removes a class.
* **`.classList.toggle('class')`**: Toggles a class on or off.

```javascript
const box = document.querySelector('.box');

// Adding an inline style
box.style.backgroundColor = 'blue';

// Toggling a class (Better approach)
box.classList.toggle('hidden');

```

---

## 🏗️ 3. Creating and Removing Elements

You can dynamically build new parts of your webpage or delete existing ones.

### ➕ Creating & Appending

1. **`document.createElement('tag')`**: Creates a new empty element.
2. **`element.appendChild(child)`**: Adds the node to the end of the specified parent's children.
3. **`element.prepend(child)`**: Adds the node to the beginning of the parent's children.

```javascript
// 1. Create a new <li> element
const newListItem = document.createElement('li');

// 2. Add text to it
newListItem.textContent = "Learn the DOM";

// 3. Append it to an existing <ul>
const ul = document.querySelector('ul');
ul.appendChild(newListItem);

```

### 🗑️ Removing Elements

* **`element.remove()`**: Removes the element entirely from the DOM.

```javascript
const oldBanner = document.querySelector('.promo-banner');
oldBanner.remove();

```

---

## 🛣️ 4. Traversing the DOM

Sometimes you need to move from one element to its relatives (parents, children, or siblings).

* **`.parentNode`** / **`.parentElement`**: Moves up to the direct parent.
* **`.children`**: Returns an HTMLCollection of direct child elements.
* **`.firstElementChild`** / **`.lastElementChild`**: Grabs the first or last child.
* **`.nextElementSibling`** / **`.previousElementSibling`**: Moves sideways to sibling elements.

```javascript
const item = document.querySelector('.active-item');
const parentList = item.parentElement;
const nextItem = item.nextElementSibling;

```

## ⚡ 5. Event Handling

Events are actions that happen in the browser (clicks, key presses, scrolling). JS allows you to listen for these and run code when they occur.

### 🎧 `addEventListener`

The modern way to attach event handlers. It takes the event type and a callback function.

```javascript
const btn = document.querySelector('#submit-btn');

btn.addEventListener('click', function(event) {
  // Prevent default form submission behavior if needed
  event.preventDefault(); 
  
  console.log('Button was clicked!');
});

```

### 🎯 Event Object

The callback function receives an `event` object (`e`) containing details about what just happened.

* **`e.target`**: The actual element that triggered the event.
* **`e.key`**: The specific key pressed (for keyboard events).

### 🫧 Event Delegation (Advanced Concept)

Instead of adding event listeners to multiple child elements (like 100 list items), add one listener to their parent element and use `e.target` to figure out which child was clicked.

```javascript
const list = document.querySelector('#todo-list');

list.addEventListener('click', function(e) {
  // Check if the clicked item was an <li>
  if (e.target.tagName === 'LI') {
    e.target.classList.toggle('completed');
  }
});

```
